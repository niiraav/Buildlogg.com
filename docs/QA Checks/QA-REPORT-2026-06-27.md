# Buildlogg — Exploratory QA Report

**Date:** Saturday, 27 June 2026  
**Tester:** Hermes Agent (automated exploratory QA)  
**Environment:** Local dev server (Vite, `localhost:5173`), Mock sign-in mode  
**Model:** GLM-5.2 (non-vision, text-based DOM inspection)  
**Scope:** Full E2E user testing — landing page, auth, onboarding, home, jobs, job detail, quote builder, settings, customers, activity, dashboard, routing  

---

## Executive Summary

| Severity | Count |
|----------|-------|
| Critical | 4 |
| High | 6 |
| Medium | 9 |
| Low | 8 |
| **Total** | **27** |

**Overall impression:** The core user journey (onboarding → create quote → track job → take payment) works well. However, the Customers feature has critical gaps — no way to add customers from the UI, and the sample customer doesn't appear in the list. The Quote Builder flow is solid end-to-end (customer form → line items → preview → send via WhatsApp/SMS/Copy all work correctly). Visual audit of 6 pages found no layout or CSS bugs.

**Testing method:** 1 lead QA agent + 6 parallel subagents across 2 rounds (3 text-based + 3 vision-attempted). 300+ tool calls total. Vision was unavailable in round 2 due to config issue (now fixed).

## Issues

### CRITICAL-1: Log Out hangs the browser indefinitely

**Severity:** Critical  
**Category:** Functional  
**URL:** `/app/settings` → Log out button  

**Description:**  
Clicking "Log out" in Settings causes the browser to become completely unresponsive. All subsequent browser navigation, snapshot, and console commands time out. The page never redirects to `/app/auth`.

**Steps to reproduce:**  
1. Sign in (mock or real)  
2. Navigate to Settings  
3. Click "Log out"  
4. Browser hangs — no navigation occurs, page is frozen  

**Expected:** User is signed out and redirected to the auth page.  
**Actual:** Browser hangs indefinitely.  

**Likely cause:** The `supabase.auth.signOut()` call may be hanging on a mock session that doesn't exist, or the sign-out promise never resolves in dev mode. The mock sign-in only sets `useAppStore.userId` and `localStorage`, not a real Supabase session — `signOut()` may block waiting for a network response that never comes.

**Evidence:** After clicking Log out via `el.click()`, all subsequent `browser_navigate`, `browser_snapshot`, and `browser_console` calls timed out (60s timeout). Server itself remained responsive (curl returned 200).
